# AI Prompt 记录

## 1. 百度首页复刻 Prompt

### 用户意图
请生成一个可运行的百度首页复刻页面，要求忠实还原搜索场景，并支持基本用户交互。

### 关键要求
- 页面要包含百度首页常见顶部导航（新闻、hao123、地图、贴吧、视频、图片、网盘、更多）。
- 搜索框应可输入关键词。
- “百度一下”按钮要能触发搜索结果显示。
- 搜索结果应包含标题、URL、摘要、快照链接，并支持分页切换。
- 搜索结果分页至少包含 3 页，且每页结果内容不同。
- 结果页应支持返回首页。
- 页面应使用可维护的 React / Next.js 代码结构，禁止直接复制百度官方源代码。

### Prompt
```
你是前端开发工程师，请基于 React/Next.js 帮我复刻一个简化版的百度首页。
页面要求：
1. 首页包含顶部导航、百度 Logo、搜索输入框、语音搜索按钮、拍照搜索按钮、百度一下按钮、热搜榜单。
2. 搜索交互要求：输入关键词后点击“百度一下”或按回车，进入搜索结果视图。
3. 结果视图要求：展示多条结果卡片，每条包含标题、URL、摘要和“百度快照”链接。
4. 分页须实现真实切换，至少 3 页结果，每页内容不同，且页码按钮可正常切换当前页。
5. 结果页上方保留搜索输入框，支持再次搜索。
6. 样式尽量贴近百度首页，但可使用本地 Tailwind / CSS 实现。

请输出完整的组件代码，并说明文件结构与核心交互逻辑。
```

## 2. 微信支付商户登录页复刻 Prompt

### 用户意图
复刻微信支付商户登录页的登录表单区域，要求实现基本验证和交互行为。

### 关键要求
- 页面包含用户名输入框、密码输入框、验证码输入框与验证码显示区域。
- 密码输入框支持显示/隐藏切换。
- 验证码图片需实现刷新逻辑，刷新后验证码字符变化。
- 登录按钮点击后进行表单校验：用户名、密码不能为空；验证码必须为 4 位；验证码输入需与当前验证码匹配。
- 登录交互应给出提示信息，比如“请输入用户名”、“验证码错误”等。
- 页面应保持微信支付登录页左右分栏布局，包含品牌说明区域。

### Prompt
```
你是前端开发工程师，请基于 React/Next.js 复刻一个微信支付商户登录页。
要求：
1. 左侧展示品牌介绍区，右侧展示登录表单。
2. 登录表单包含用户名、密码、验证码输入和登录按钮。
3. 密码输入框支持点击切换明文/密文。
4. 验证码显示为可刷新字符块，每次刷新生成新的 4 位验证码。
5. 点击登录时校验：用户名不能为空、密码不能为空、验证码不能为空、验证码必须为 4 位且与当前验证码一致。
6. 校验失败时给出对应提示。
7. 成功后显示“登录成功，正在跳转...”之类的提示。

请输出完整组件代码与交互逻辑实现。
```

## 3. 知乎登录页复刻 Prompt

### 用户意图
生成知乎登录页复刻，支持手机号/密码登录切换和验证码交互。 

### 关键要求
- 页面左侧展示品牌与产品价值信息，右侧为登录表单。
- 登录表单应支持“手机号登录”和“密码登录”两个 Tab 切换。
- 手机号登录包含手机号输入、验证码输入、发送验证码按钮，以及倒计时功能。
- 密码登录包含账号输入、密码输入和密码显隐切换。
- 登录按钮点击后应模拟登录交互，例如 loading 状态与成功提示。
- 页面需实现登录方式切换、验证码发送、倒计时、密码显隐等交互。

### Prompt
```
你是前端开发工程师，请基于 React/Next.js 复刻一个知乎登录页。
要求：
1. 页面包含左侧品牌展示区和右侧登录表单区。
2. 登录表单支持“手机号登录”和“密码登录”两种方式切换。
3. 手机号登录页面包含手机号输入、验证码输入框、发送验证码按钮和倒计时逻辑。
4. 密码登录页面包含账号/手机号输入、密码输入和明文/密文切换按钮。
5. 点击发送验证码后启动倒计时，期间按钮不可重复点击。
6. 点击登录后显示 loading 状态，并在完成后展示登录成功提示。

请输出完整 React 组件实现与交互说明。
```

## 4. 一致性评估 Prompt

### 用户意图
对比原始网页与复刻产物，自动生成一致性评估报告，输出可量化评分。 

### 关键要求
- 评估维度：视觉一致性、功能一致性、交互一致性。
- 每个维度下再细分具体指标：DOM结构一致性、CSS属性匹配、布局相似度、色彩还原度、功能覆盖率、功能正确率、数据完整性、交互还原度、反馈一致性、状态管理。
- 输出必须为 JSON，不能包含多余文本。
- JSON 结构须包含：`visual`、`functional`、`interaction`、`overall`、`summary`。

### Prompt
```
你是网页复刻质量评估专家。请对比原始页面和复刻页面，从视觉、功能、交互三个维度进行量化评估。
请严格返回 JSON，不要输出其他文本。
返回格式：
{
  "visual": {
    "overall": <0-100>,
    "dom_structure": <0-100>,
    "css_match": <0-100>,
    "layout_similarity": <0-100>,
    "color_accuracy": <0-100>
  },
  "functional": {
    "overall": <0-100>,
    "feature_coverage": <0-100>,
    "feature_correctness": <0-100>,
    "data_integrity": <0-100>
  },
  "interaction": {
    "overall": <0-100>,
    "interaction_fidelity": <0-100>,
    "feedback_consistency": <0-100>,
    "state_management": <0-100>
  },
  "overall": <0-100>,
  "summary": "<50字以内评估总结>"
}

原始页面描述：<描述原始官网页面结构与核心功能>。
复刻页面描述：<描述复刻产物实现的功能与交互>。

请给出各项评分及简短总结。
```
